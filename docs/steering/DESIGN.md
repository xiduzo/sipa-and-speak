## Design System Name: The Academic Atrium

**Creative north star: "The Curated Conversation"**

The UI feels like a high-end university lounge ‚Äî warm, intellectual, vibrantly social. We reject clinical grids in favour of intentional asymmetry: overlapping elements, a high-contrast type scale, and deliberate white space that breathes between ideas. Every screen should feel bespoke, not templated.

## Design Principles

1. **Real-world warmth** ‚Äî Surfaces, colours, and textures evoke a caf√©, not a dashboard. Every screen should feel like a place worth spending time in.
2. **Asymmetry as movement** ‚Äî Reject perfect centred stacks. Overlapping elements, offset cards, and contrast jumps in type create a sense of flow that mirrors natural conversation.
3. **Depth without lines** ‚Äî Hierarchy is expressed through surface-tier shifts and ambient shadow, never 1px borders. Boundaries should be felt, not seen.
4. **Typography leads** ‚Äî A large headline followed by a direct jump to body creates a magazine aesthetic. Type hierarchy does the heavy lifting; decoration is secondary.
5. **Generous space** ‚Äî When in doubt, add more white space. A busy screen is a failure of editing, not a lack of content.

### Shadow / Elevation

No standard drop shadows. Two allowed forms only:

- **Ambient glow** ‚Äî `on-surface` at 6% opacity, 32px blur. For floating action buttons.
- **Ghost border fallback** ‚Äî `--border` at 15% opacity. Use only when a boundary is accessibility-required.

### Glassmorphism

For floating navbars and quick-action modals: `surface` at 80% opacity + `backdrop-blur: 20px`. For overlays and bottom sheets: `surface-variant` + `backdrop-blur: 12px`.

## The "No-Line" Rule

**1px solid borders for sectioning are strictly prohibited.** Define boundaries through background color shifts between `surface-container` tiers. If a boundary is required for accessibility, use `--border` at 15% opacity (the "Ghost Border"). It must be felt, not seen.

## Hero CTA Gradient

Main hero CTAs use a subtle linear gradient from `primary` ‚Üí `primary-container` (not a flat fill). Adds dimension and "soul" to the primary brand moment.

## UI States Required

Every interactive feature must cover all of these:

- **Default** ‚Äî base state
- **Loading** ‚Äî async in progress; use `Skeleton` component (`packages/ui/src/components/skeleton.tsx`)
- **Error** ‚Äî inline message below the field, never a blocking modal unless data is at risk
- **Empty** ‚Äî descriptive copy + a primary CTA; never a blank screen
- **Disabled** ‚Äî `opacity-50`, `pointer-events-none`
- **Focus** ‚Äî 2px visible ring using `--ring` token; never remove focus outlines

## Accessibility Standards

Baseline: **WCAG 2.1 AA**

- Minimum contrast: 4.5:1 for body text, 3:1 for large text and UI components
- All interactive elements keyboard-navigable with a visible focus indicator
- All images have descriptive `alt` text; decorative images use `alt=""`
- Color is never the sole means of conveying information ‚Äî pair color cues with icons or text
- Form inputs have associated `<label>` elements (not placeholder text alone)
- ARIA states used on interactive components: `aria-invalid`, `aria-expanded`, `aria-disabled`
- Background textures (linguistic characters) must use `aria-hidden="true"`

## Responsive Behavior

This is a **mobile-first** product. The native Expo app is the primary surface; the web app is secondary.

| Breakpoint | Width | Layout behaviour |
| --- | --- | --- |
| Mobile (native) | Expo / device width | Single-column, full-bleed surfaces; bottom sheet navigation |
| Mobile web | < 640px | Single-column; matches native layout closely |
| Tablet / Desktop web | ‚â• 768px | Centred single-column max-width container; side padding increases |

## Do's and Don'ts

**Do:**
- Use asymmetrical layouts ‚Äî let images overlap two different surface containers to create depth
- Use the full spacing scale generously; default to `spacing-12` or `spacing-16` for editorial sections
- Use Script Patterns ‚Äî subtle 5% opacity linguistic characters as background textures to celebrate the multilingual community

**Don't:**
- Use 1px solid borders ‚Äî they kill the premium feel
- Use pure white (`#ffffff`) as a background ‚Äî always use `surface` (`#fff8f6`)
- Use standard Material Design drop shadows ‚Äî keep them diffused, tinted, and ambient
- Crowd the interface ‚Äî if a screen feels busy, remove elements or increase surface-tier contrast
