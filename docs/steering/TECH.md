## Architecture

- **Pattern:** Monorepo with clear package boundaries — apps consume packages, packages do not consume apps
- **Module structure:**
  - `apps/web` — React SPA (Vite, TanStack Router)
  - `apps/native` — Expo mobile app (expo-router, file-based routing)
  - `apps/server` — Hono HTTP server, mounts tRPC and Better-Auth handlers
  - `packages/api` — All tRPC routers (`profile`, `matching`, `venue`, `meetup`, `chat`, `healthCheck`)
  - `packages/db` — Drizzle schema, migrations, seed scripts
  - `packages/auth` — Better-Auth configuration (email OTP, Polar plugin, Expo native)
  - `packages/ui` — Shared React components and Tailwind tokens
  - `packages/env` — T3 env validation for type-safe environment variables
  - `packages/config` — Shared TypeScript config base
- **Data flow:** Client → tRPC client (typed) → Hono server → tRPC router → Drizzle query → PostgreSQL
- **Auth flow:** Better-Auth handles session management server-side; native app uses the Expo-specific auth plugin

## Key Constraints

- **Bun only** — use `bun install`, `bun run`; never `npm` or `yarn`; commit `bun.lock`
- **All API changes via tRPC** — no ad-hoc REST endpoints; new server functionality goes in `packages/api/src/routers/`
- **Schema changes via Drizzle migrations** — never modify the DB directly; always `bun db:generate` then `bun db:migrate`
- **Auth via Better-Auth only** — no custom session or credential logic outside `packages/auth`

## Commands

```bash
# Install
bun install

# Start local database
bun db:start

# Dev (all apps)
bun dev

# Dev individual apps
bun dev:web
bun dev:native
bun dev:server

# Type-check all packages
bun check-types

# Build all apps
bun build

# Database: generate migration from schema changes
bun db:generate

# Database: apply migrations
bun db:migrate

# Database: push schema directly (dev only)
bun db:push

# Database: open Drizzle Studio
bun db:studio
```

## Code Conventions

- **Naming:** PascalCase React components, camelCase functions and variables, kebab-case file names
- **File structure:** Feature-based folders within each app; shared logic extracted to `packages/`
- **Imports:** Use package workspace aliases (e.g. `@repo/db`, `@repo/api`) — no deep relative `../../` imports across package boundaries
- **Routers:** One file per domain in `packages/api/src/routers/`; all routers registered in `packages/api/src/routers/index.ts`
- **Schema:** Domain tables in `packages/db/src/schema/sip-and-speak.ts`; auth tables in `packages/db/src/schema/auth.ts`
- **Test files:** No test suite currently exists — write new code with pure functions and clear side-effect boundaries to enable future coverage
- **Native styling:** Prefer NativeWind `className` over `style={{}}` inline objects. Use token-based utilities (`text-foreground`, `bg-background`, `font-manrope`, `font-manrope-bold`, `bg-brand-gold`, etc.) defined in `apps/native/global.css`. Reserve the `style` prop for genuinely dynamic values that can't be expressed as a className string (e.g. pressed/disabled state colors, conditional `borderColor`). Never declare hardcoded color constants (BG, INK, MUTED) when a design token class exists.

## Architecture Decision Records

| Decision | Rationale | Date |
| --- | --- | --- |
| tRPC over REST | End-to-end type safety between server and both web + native clients without code generation | 2025 |
| Better-Auth over Auth.js | Native Expo support and email OTP plugin; Auth.js lacked a stable React Native adapter | 2025 |
| Drizzle over Prisma | Lighter runtime, SQL-close query API, better Bun compatibility | 2025 |
| Turborepo | Parallel builds and caching across monorepo packages with minimal config | 2025 |

## Known Pain Points

- **No test suite** — zero automated coverage; prioritise writing pure, side-effect-free functions so coverage can be added without major refactoring
- **Native auth flow** — the Expo + Better-Auth integration is non-standard; changes to `packages/auth` or `apps/native/lib/auth-client.ts` require end-to-end manual testing on both iOS and Android
